【任务】用 C++11 实现一个金融因子表达式解析器。

【因子表达式语言】

1. 逐元素运算符（element-wise，每个时间/股票独立计算）：

2. 时间序列函数（沿每只股票的时间轴纵向计算）：
   delay(x, d)         -- x 滞后 d 个时间点
   delta(x, d)         -- x - delay(x,d)
   ts_sum(x, d)        -- 过去 d 天的滚动和
   ts_mean(x, d)       -- 过去 d 天的滚动均值
   ts_std(x, d)        -- 滚动标准差
   ts_corr(x, y, d)    -- 滚动相关系数
   ...（按需添加）

3. 横截面函数（在同一时间点所有股票上计算）：
   rank(x)             -- 横截面排名，输出缩放到 0~1

【输入】

【解析目标】

## 1.  核心数据布局与维度规范 (Panel Data)

- **扁平大数组**：所有矩阵数据（如 `Close`、中间结果、最终输出）统一存储于一维连续的 `std::vector<float>` 中。
- **形状与存储**：矩阵大小固定为 `T * N`（T 为历史总交易日天数，N 为全市场总股票标的数），并严格采用**行优先（Row-Major）**存储。
- **索引映射公式**：在 C++ 算子内部，定位第 t 天、第 n 只股票的内存下标必须严格使用：
  $$\text{index} = t \times N + n$$ 
  其中 0 ≤ t < T 且 0 ≤ n < N。

## 2.  缺失值与未上市股票处理 (NaN 传播规则)

- **空洞填充**：对于未上市、停牌或退市的交易日，矩阵对应位置的数据统一使用 `std::numeric_limits<float>::quiet_NaN()` 填充。

- 算子鲁棒性

  ：任何算子在遍历数组时，必须先用 

  ```
  std::isnan(val)
  ```

   拦截。

  - **时序算子（纵向）**：忽略窗口内的 `NaN`，仅对有效数字求最大值；若整个窗口全为 `NaN`，则结果填入 `NaN`。
  - **截面算子（横向）**：排序时，直接剔除当天值为 `NaN` 的股票，动态调整当天的有效股票分母 M。未参与排序的股票，其结果保持 `NaN`。
  - **四则运算**：任何数字与 `NaN` 发生运算，结果一律传播为 `NaN`。

## 3. 内存管理与生命周期 (零拷贝传递)

- **避免值拷贝**：由于 `T * N` 数组可能较大，在 ExprTk 自定义算子之间传递矩阵时，禁止发生显式的 `vector` 复制。
- **智能指针包装**：所有算子的返回值和中间变量，统一在堆上开辟，并使用 **`std::shared_ptr<std::vector<float>>`** 进行生命周期管理。
- **自动释放**：利用智能指针的析构特性，确保内层算子（如 `Ts_Max`）产生的临时大数组在被外层算子（如 `Rank`）消费完毕后，内存能够**立即自发释放**。在对应位置留下 `// TODO: 后续引入 MatrixPool 进一步优化` 的注释。

## 4.  算子体系设计要求

请在 ExprTk 框架下，利用 `exprtk::igeneric_function<double>` 注册并串联以下算子：

- **`Ts_Max(Matrix, Window)`（时序纵向算子）**：固定股票 n，沿着时间轴 t （纵向）执行滑动窗口求最大值，返回一个全新的 `T * N` 矩阵指针。

- **`Rank(Matrix)`（截面横向算子）**：固定交易日 t，沿着全市场股票 n（横向）执行绝对名次排序，并归一化为 $\frac{\text{绝对名次}}{\text{当天有效总数 } M}$ 的百分比排名（数值在 0.0 ~ 1.0 之间），返回最终结果指针。
